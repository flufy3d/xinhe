# 🧠 心核  架构蓝图：大一统快慢交替记忆网络
*(Grand Unified Fast-Slow Alternating Memory Network)*

### 🌌 核心设计哲学

**"统一海马体，分层新皮层；快慢分流，昼夜交接。"**

模型由三层记忆载体协同：**永久冻结的 Base 基座**承载预训练知识；**全局唯一的海马体 W**（Attention 侧的快速可塑矩阵）承载当日工作记忆，依靠 Delta Rule 在 L2 归一化的 key 空间中自动寻找正交槽位、按内容寻址精准覆写；**per-layer 全秩并联的 Memory MLP**（新皮层扩充神经元）承载跨日长期记忆，是真正长出的新皮层组织，而非对 Base 的低秩扰动。

记忆的演化由一条极简法则驱动：**Delta Rule 内容驱动覆写**——k_proj 学会把不同语义路由到不同正交槽位，v_proj 学会写入要被检索的内容，β 学会区分该写多少。在统一的 W 内，槽位可以被同 key 的新内容自然覆写，无需人工切割记忆类型，无需 RoPE 时序探针，无需显式衰减常数。

记忆的流转遵循昼夜节律：白天海马体高速运转、新皮层冻结观摩；夜晚海马体冻结，通过分段重建蒸馏将当日存活的高纯度记忆永久烙印进新皮层；翌日 W 清零，以更渊博的新皮层开启新一天。"快慢交替"的"快"指当日海马体 W，"慢"指跨日 Memory MLP；快慢之间通过夜间蒸馏交接。

### 🧮 核心数学原理 (The Mathematical Heart)

每一个推理步/微调步,记忆的演化由两组极简方程决定:

**1. 大一统状态演化方程(纯 Delta Rule):**
$$W^{(t)} = W^{(t-1)} + \beta_t \cdot (v_t - W^{(t-1)}k_t) \otimes k_t^T$$
* **物理意义:** 把现有 W 在 $k_t$ 方向上的旧值减掉,再以强度 $\beta_t$ 写入新值 $v_t$。等价于在 $k_t$ 这条正交方向上做一次自适应覆写。
* **k 必须 L2 归一化:** 抑制 W 范数爆炸,同时让 Delta Rule 的"正交槽位"几何学清洁。

**2. 极简读取方程:**
$$\text{read} = q_t \cdot W^{(t)\,T}$$
* **物理意义:** $\mathcal{O}(1)$ 常数级复杂度。q 与 W 的列做线性匹配,正确的语义会依靠其在 k 空间的位置自然涌现。读侧 q 同样 L2 归一化以匹配写侧。

### 🛠️ 工程实现细节 (Implementation Checklist)

**1. 网络结构修改(挂载于 Attention 层)**
* **单一记忆张量:** `W`,形状 `(Batch, num_heads, head_dim, head_dim) = (B, 16, 128, 128)`。
* **写侧投影:**
  * `k_proj: nn.Linear(hidden, n_heads * head_dim, bias=False)` — 全局共享,作用于 final content_output
  * `v_proj: nn.Linear(hidden, n_heads * head_dim, bias=False)` — 同上
  * `beta_proj: nn.Linear(hidden, n_heads, bias=True)` — per-head 写强度
* **读侧投影(per-layer):**
  * `q_projs[ℓ]: nn.Linear(hidden, n_heads * head_dim, bias=False)` — 每个 hook 层一份
  * `o_projs[ℓ]: nn.Linear(n_heads * head_dim, hidden, bias=False)` — 同上
  * `read_scale: nn.Parameter(torch.tensor(-3.0))` — 标量,σ(read_scale) 控制 W 输出强度

**2. LoRA 挂载配置(中央化于 `configs/base.yaml`)**
* **Rank:** 16
* **Alpha:** 32
* **Target Modules:** `["q_proj", "k_proj", "v_proj", "o_proj"]`
    * **绝对不能遗漏 `k_proj`** — 它是模型学会寻址正交槽位的关键。
    * 线性 attention 层(`in_proj_qkv` / `out_proj`)实测无收益,已不挂 LoRA。
* **唯一定义点:** `base.yaml`,所有子 config 不再覆盖 target_modules。

**3. 写 kernel 后端策略(`xinhe/model/delta_kernel.py`)**

`delta_rule_write` 提供两个等价实现:
* **torch `_delta_parallel`**: chunkwise 并行闭式解,内部 fp32 求三角系统。
* **FLA `chunk_delta_rule`**: Triton kernel,bf16 输入。

`Hippocampus.write_from_content` 按 `self.training` 自动派发:

| 模式 | 后端 | 理由 |
|---|---|---|
| `model.train()` | **强制 torch** | FLA Triton backward 在 bf16 上引入 5–25% 梯度幅度误差,优化器收敛劣化 |
| `model.eval()` | **配置 backend**(默认 `auto` → Linux+CUDA 用 FLA) | FLA forward 与 torch fp32 误差 < 0.5%,可享受 Triton 加速 |

**4. 新皮层并联扩充模块(Memory MLP,per-layer 全秩 SwiGLU)** *(Phase 2,尚未实装)*

每个 Transformer Block 在 Base MLP 旁**并联一个全新的、独立的 SwiGLU 模块**,与 Base MLP 同形态但**参数完全独立**:
$$W_{gate}^{mem,(\ell)}, W_{up}^{mem,(\ell)} \in \mathbb{R}^{d_{model} \times d_{mem}}$$
$$W_{down}^{mem,(\ell)} \in \mathbb{R}^{d_{mem} \times d_{model}}$$

最终 MLP 层输出为两条通路之和:
$$\text{MLP}_{out}^{(\ell)}(x) = \underbrace{\text{MLP}_{base}^{(\ell)}(x)}_{\text{冻结}} + \underbrace{\big(\text{SiLU}(x W_{gate}^{mem,(\ell)}) \odot x W_{up}^{mem,(\ell)}\big) W_{down}^{mem,(\ell)}}_{\text{新增神经元}}$$

* **本质**:这是对基座的**结构性扩充**,不是低秩扰动。Base MLP 的三个矩阵从未被触碰。
* **$d_{mem}$ 推荐**:与 Base intermediate 同量级或减半(例如 Base intermediate=8192 时取 $d_{mem}=4096$)。
* **初始化**:`W_down` 初始化为 0,保证训练初期 Memory MLP 输出为零,不破坏 Base 行为;`W_gate`、`W_up` 用 Base MLP 对应矩阵的初始化方案。

### 🌗 系统生命周期 (The Day/Night Lifecycle)

这套架构天然复刻哺乳动物的【海马体-新皮层】记忆流转机制:

* **☀️ 白天模式 (Daytime Inference & Short-term):**
    * 激活 Attention 层的 LoRA。
    * Memory MLP 参与前向预测,但**冻结不更新**。
    * $W$ 矩阵作为高维海马体高速运转。Delta Rule 自动把同 key 的新内容覆盖旧值;不再被检索的内容会因为槽位被复用而自然淡出,无需显式时间衰减。

* **🌙 夜晚模式 (Sleep Consolidation):** *(详见下节)*

* **🌅 翌日重启:**
    * 清空 $W$ 矩阵,以更加渊博的基座权重开启新一天的对话。

## 🌙 夜晚模式:分段重建蒸馏机制 (Sleep Consolidation)

### 一、核心哲学

睡眠的本质是**记忆载体的迁移**:白天可塑的海马体(W + Attention LoRA)所承载的当日新知识,必须被永久烙印进新皮层(Memory MLP 全秩神经元),第二天才能在 W 清零后依然保有这些经验。

夜晚训练遵循**严格阶段分离**原则:
* 海马体侧(Attention LoRA、k_proj/v_proj/q_projs/o_projs/beta_proj/read_scale):**全部冻结**
* Base 模型全部权重:**永久冻结**
* 新皮层侧(Memory MLP 的 `W_gate^mem` / `W_up^mem` / `W_down^mem`):**唯一可训练对象**
* W 矩阵:**不更新参数**,仅作为 Teacher 通路的状态载体被重建

### 二、白天侧的归档机制(为夜晚准备数据)

为了让夜晚能精确复现白天的 W 演化轨迹,白天必须按窗口归档:

**1. 窗口划分**
连续对话每满 K 轮(推荐 20–50 轮)或累计 ≤ 8K tokens 即封口为一个窗口 $w$。闲置超 30 分钟也强制封口。

**2. 检查点 + 输入缓冲**
窗口起始保存 W 全状态:
$$\text{CKPT}_w = W_w^{start} \in \mathbb{R}^{B \times 16 \times 128 \times 128}$$
窗口内累积输入序列:
$$\text{Buf}_w = \{ x_{w, 0:T_w}, \text{masks}, \text{labels} \}$$

**3. 存储成本**
单 CKPT:$16 \times 128 \times 128 \times 2$ bytes (fp16) ≈ **512 KB**
一天 10–20 窗口 ≈ **5–10 MB**。仍可忽略。

**4. 一天归档**
$$\mathcal{R}_{today} = \{ (\text{CKPT}_w, \text{Buf}_w) \}_{w=1}^{N_w}$$

### 三、Teacher / Student 配置

| 角色 | Base 权重 | Attention LoRA | Hippocampus 写读投影 | W 矩阵状态 | Memory MLP |
|---|---|---|---|---|---|
| **Teacher** | 冻结 | 打开,当前权重,**冻结** | 冻结 | 由 CKPT 重建轨迹并读取 | 当前权重(旧),冻结 |
| **Student** | 冻结 | 打开,当前权重,**冻结** | 冻结 | **关闭(强制 read 输出为 0)** | **可训练**(更新全秩权重) |

**关键**:Student 必须打开 Attention LoRA,并且保持冻结。Student 唯一需要关闭(强制为 0)的,只有从 W 矩阵读取知识的那一步($\text{Read}(W) \rightarrow 0$),必须靠 per-layer 全秩 Memory MLP 自己复现 Teacher 完整通路(含 W 检索)的行为。这是真正的"知识从海马体蒸馏到新皮层"——新皮层在长出新的神经元来承载这些知识。

### 四、单窗口蒸馏流程

**Step A — Teacher 重建 W 轨迹**
载入 $W_w^{start}$,沿 $x_{w, 0:T_w}$ 跑 Teacher 前向。逐 token 严格按白天规则更新 W:
$$W^t = W^{t-1} + \beta_t \cdot (v_t - W^{t-1}k_t) \otimes k_t^T$$
同时记录每 token:
* logits $P_t^{T}$
* 各层中间隐藏态 $H_t^{T,(\ell)}$

**Step B — Student 前向**
同输入跑 Student(W 读取关、Attention LoRA 开(冻结)、Memory MLP 开),得 $P_t^{S}, H_t^{S,(\ell)}$。

**Step C — 蒸馏损失**
$$\mathcal{L}_w = \mathcal{L}_{KL}(P^{T} \parallel P^{S}) + \alpha \sum_{\ell} \text{MSE}(H^{T,(\ell)}, H^{S,(\ell)})$$

* **KL 项**:行为蒸馏,让 Student 输出分布对齐 Teacher
* **MSE 项**:特征对齐,强迫每层中间表征贴近,缓解纯 KL 的"输出对但中间漂移"问题

梯度只流向 Memory MLP 的 `W_gate^mem` / `W_up^mem` / `W_down^mem`(全秩矩阵,全参数训练)。

### 五、关键性质

| 性质 | 说明 |
|---|---|
| **窗口独立** | 每窗口从真实 $W^{start}$ 起步,无跨窗口误差累积 |
| **轨迹完整** | 每 token 都有 Teacher 真实贡献,蒸馏覆盖 W 整条演化曲线 |
| **可并行** | 窗口间无依赖,多卡可并行训练 |
| **可恢复** | 磁盘掉电后从 CKPT + Buf 严格复现任何窗口 |
| **存储极轻** | 一天归档 < 10 MB,长期保留无压力 |
| **结构性扩充** | Memory MLP 是真神经元,Base 不被任何形式扰动 |

### 六、防长期遗忘机制

固定规模的 Memory MLP 在每日新知识冲刷下必然趋向饱和。核心防护策略:

**混合 Batch 策略**
* 70% 当日窗口 $\mathcal{R}_{today}$
* 30% 历史窗口 $\mathcal{R}_{history}$(reservoir sampling 维持长期分布)

### 七、醒后状态切换 (🌅 翌日重启)

Sleep 结束时执行:
1. **持久化** Memory MLP 全部三个矩阵的新权重到磁盘
2. **清空** $W \leftarrow 0$(当日海马体记忆已迁移完毕)
3. **保留** Attention LoRA 与 Hippocampus 写读投影的状态延续到第二天(这些是"如何使用海马体"的元能力,跨日累积训练)
4. **归档** 当日 $\mathcal{R}_{today}$ 进入历史池

### 八、关键超参

| 参数 | 推荐值 | 说明 |
|---|---|---|
| K(窗口轮数)| 20–50 | 或 ≤ 8K tokens 封口 |
| $d_{mem}$(Memory MLP 中间维)| Base intermediate 的 50%–100% | 决定长期容量上限 |
| $\alpha$(特征对齐权重)| 0.1–1.0 | KL 已收敛后再加大 |
| 历史回放比例 | 30% | 随容量饱和度上调至 50% |
| Memory MLP 学习率 | $1\text{e-}4$ | AdamW |
| KL 蒸馏温度 | 2–4 | 软化分布利于学习 |
| 单窗口 epoch 数 | 1–3 | 过拟合风险 vs 蒸馏充分度 |

### 九、监控指标

| 指标 | 期望 / 阈值 |
|---|---|
| Sleep KL loss 收敛曲线 | 单调下降 |
| Student vs Teacher 每层 CKA | 训练后 > 0.8 |
| 当日窗口 retention(即时验证)| > 95% |
| 保留集 retention(历史验证)| 周度,下降 > 5% 告警 |
| Memory MLP 跨层梯度范数均衡度 | 浅层不能 starve |
| Memory MLP 各神经元激活稀疏度 | 监控容量饱和征兆(激活率持续上升 = 接近饱和)|
