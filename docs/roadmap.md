## 阶段 A：基础验证

### 1. 基线

**目标**：MiniMind 正常加载能聊天

**操作**：
- 克隆 MiniMind 仓库，下载预训练权重
- 加载 MiniMindBackbone，不加 StatePlugin
- 直接聊天验证

**通过标准**：中文对话流畅，回答合理

### 2. 空状态

**目标**：加 StatePlugin (scale≈0) 后仍能正常聊天

**操作**：
- 加载完整 XinheModel（backbone + plugin），不训练
- state_scale_init = -5.0 → sigmoid ≈ 0.007
- 直接聊天验证

**通过标准**：聊天质量与基线无明显差异

**验证了什么**：三层安全保护有效（LoRA 零初始化 + 状态近零 + 渐进 scale）

## 阶段 B：记忆涌现

### 3. 1轮记忆

**目标**：训练后能记住上一轮说的话

**操作**：
- 用多轮对话数据训练（SFT 数据 / 合成记忆数据）
- 聊天测试："我叫张三" → 下轮问 "我叫什么？"

**通过标准**：能正确回答上一轮提供的信息

**验证了什么**：state 能在 segment 间传递信息

### 4. 多轮记忆

**目标**：跨多轮保留信息

**操作**：
- 第 1 轮告诉信息
- 第 5 轮、第 10 轮、第 15 轮分别询问

**通过标准**：
- 5 轮后准确率 > 80%
- 10 轮后准确率 > 50%
- 测试 retention 曲线（准确率 vs 距离）

**验证了什么**：state + gate 能实现长期保留

### 5. 信息覆写

**目标**：能更新旧信息

**操作**：
- "我在北京" → 几轮后 "我搬到上海了" → 问 "我在哪？"

**通过标准**：回答最新信息（上海），不是旧信息（北京）

**验证了什么**：gate 的动态部分在工作——能根据内容决定覆写

### 6. Wipe 对比

**目标**：证明模型确实依赖状态，而不是靠 backbone 本身"记住"

**操作**：
- 正常聊天，告诉信息
- `/wipe` 清除状态
- 问同样的问题

**通过标准**：wipe 后回答不出来（或明显下降）

**验证了什么**：信息确实存在 state 里，不是巧合

### 7. 时间尺度

**目标**：快慢变量自发涌现

**操作**：
- 训练完后检查 gate_bias 分布
- 用线性探针分析各维度存储的信息类型

**通过标准**：
- gate_bias 分布呈双峰或多峰（不是单峰）
- 慢变量维度能解码长期信息（名字、偏好）
- 快变量维度存储近期信息（最近话题）

**验证了什么**：多时间尺度确实从统一结构中涌现

## 阶段 C：在线学习

### 8. Sleep — 状态压缩

**目标**：sleep pass 有效压缩状态

**操作**：
- 长对话后 `/sleep`
- 对比 sleep 前后的状态统计

**通过标准**：
- 状态有效秩下降（SVD 分析）
- 状态范数变化（弱信号衰减）
- sleep 后 retention 不下降（甚至提升）

**验证了什么**：无输入的自整理能压缩信息

### 9. Sleep — 权重更新

**目标**：sleep 在线学习有效，对话信息固化进权重

**操作**：
- 聊天一天 → `/sleep`（触发权重更新）
- 第二天加载 .pt → 验证还记得昨天的事
- 对比：不 sleep 直接加载 state vs sleep 后加载

**通过标准**：
- sleep 后 plugin 权重有变化（weight diff > 0）
- 第二天能回忆昨天的对话内容
- sleep 后的 retention 优于不 sleep

**验证了什么**：两套记忆系统在工作（state=短期，权重=长期）

### 10. 灵魂分化

**目标**：不同用户的 AI 变得不同

**操作**：
- 从同一个预训练 .pt 出发
- 用户 A 聊技术话题，经历多次 sleep
- 用户 B 聊生活话题，经历多次 sleep
- 用同样的问题测试两个 .pt

**通过标准**：
- 两个 .pt 的权重 diff 显著
- 对同样问题的回答风格/内容不同
- 各自记住各自用户的信息

**验证了什么**：.pt 确实在分化，"灵魂"在成长

### 11. 消融实验

逐个关闭组件，测量影响：

| 消融项 | 关闭方式 | 预期影响 |
|--------|---------|---------|
| state 数量 | n_state: 32→16→8→4 | retention 下降 |
| gate 静态偏置 | gate_bias 全零 | 时间尺度分化消失 |
| gate 动态部分 | 去掉 gate_proj | 覆写能力下降 |
| sleep（状态） | 不做 sleep pass | 长距离 retention 下降 |
| sleep（权重） | sleep 时不更新权重 | 跨天记忆消失 |
| LoRA | lora_rank: 16→8→4→0 | backbone 无法学会读写状态 |
| TBPTT 窗口 | tbptt_steps: 4→2→1 | 跨 segment 梯度断裂 |
| sleep_lr | 1e-5→1e-4→1e-6 | 学习速度 vs 稳定性 |

## 未来方向（验证成功后）

### 近期
- **Qwen-7B 迁移**：实现 QwenBackbone，重训 LoRA + Plugin
- **分层 state**：快 state（小，每轮更新）+ 慢 state（大，sleep 时更新）

### 远期
- **State-level RAG**：检索历史 state 快照，融合进当前 state → 真正无限记忆
- **多 AI 社交**：不同 .pt 之间交换 state → AI 之间的"交流"
- **state 遗传**：从一个成熟 .pt 初始化新 AI → 类似"培养"
